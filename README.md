## 📌 סקירה כללית
**עוזר ה-Chef-AI** הוא סוכן חכם מבוסס אירועים (Event-Driven), שנועד לשמש כנציג תמיכה טכנית וארכיטקט תוכנה עבור פרויקט "Chef-AI".

בניגוד למערכות RAG (Retrieval-Augmented Generation) מסורתיות שמסתמכות אך ורק על חיפוש סמנטי, מערכת זו מיישמת **ארכיטקטורת Hybrid RAG**. היא מנתבת בקשות בצורה חכמה בין מסד נתונים וקטורי בלתי מובנה (Pinecone) לבין מאגר ידע מובנה ומדויק (JSON), מה שמבטיח דיוק מרבי, שליפת רשימות מלאות ואפס הזיות (Hallucinations).

---

## ✨ תכונות עיקריות וארכיטקטורה

### 1. 🧠 נתב LLM חכם (ה"מוח")
הלב של המערכת הוא תהליך זרימה (`Workflow`) מותאם אישית שנבנה עם LlamaIndex. לפני ביצוע חיפוש כלשהו, מודל השפה מנתח את כוונת המשתמש ומנתב את הבקשה ליעד הנכון:
* **`JSON_RULES` / `JSON_DECISIONS`:** לבקשות של רשימות מקיפות (למשל, "הצג את כל חוקי העיצוב"), הנתב עוקף את החיפוש הווקטורי ושולף את הנתונים המובנים ישירות מתוך קובץ ה-JSON.
* **`PINECONE`:** לשאלות עומק והסברים של "איך" או "למה" (למשל, "למה בחרנו ב-MongoDB?"), הנתב מפנה את השאלה ל-Pinecone להבאת הקשר סמנטי.
* **`CHAT`:** לשיחות חולין או שאלות מחוץ להקשר (למשל, "מה השעה?"), המערכת לא מבצעת חיפוש כלל כדי למנוע "הרעלת" הקשר (Context Poisoning).

### 2. 📊 חילוץ נתונים מובנה אוטומטי (ETL)
המערכת כוללת סקריפט מתקדם (`extract_data.py`) שסורק מסמכי Markdown, משתמש ב-Chat API של חברת Cohere יחד עם `PydanticOutputParser`, ואוכף סכמת JSON נוקשה כדי לחלץ חוקים והחלטות. מנגנון זה פותר את מגבלת ה-RAG המסורתית שמתקשה בשליפת רשימות מלאות וארוכות.

### 3. 🛡️ חוקי ברזל (מניעת הזיות)
המערכת מוגדרת עם הוראות מערכת (System Instructions) קפדניות במיוחד. היא כבולה מתמטית להקשר שסופק לה. אם משתמש שואל על מזג אוויר, פוליטיקה או תכונות שלא תועדו, הסוכן בולם זאת ועונה באלגנטיות: *"המידע אינו מופיע במסמכי המערכת"*.

### 4. 🔄 זיכרון שיחה (Conversational Memory)
הסוכן מנהל זיכרון פנימי (`ChatMemoryBuffer`), המאפשר למשתמשים לנהל שיחה זורמת ולשאול שאלות המשך בטבעיות. הנתב ומייצר התשובות עובדים יחד כדי להבין את ההקשר (למשל, לשאול "מה מתוך הרשימה הזו הכי חשוב?" מיד אחרי קבלת רשימה מה-JSON).

## 🛠️ טכנולוגיות (Tech Stack)
* **תשתית (Framework):** LlamaIndex (מבוסס Workflows & Events)
* **מודל שפה והטמעות (LLM & Embeddings):** Cohere (מודל `command-r-08-2024` לשיחה ו-`embed-multilingual-v3.0` להטמעות וקטוריות)
* **מסד נתונים וקטורי:** Pinecone
* **אימות נתונים:** Pydantic (לפירוק JSON ואכיפת מבנה קפדני)
* **ממשק משתמש (UI):** Gradio (כולל התאמות CSS מיוחדות)

---

## ⚙️ איך זה עובד (זרימה מבוססת אירועים)
1. **`StartEvent`** ➡️ מאמת את תקינות הקלט (אורך מינימלי).
2. **`ValidationEvent`** ➡️ מפעיל את צעד הנתב (`router`).
3. **`RouterEvent`** ➡️ ה-LLM מחליט מהו מקור המידע הנכון (`JSON`, `PINECONE`, או `NONE`).
4. **`RetrievalEvent`** ➡️ שולף את מודלי ה-Pydantic המובנים או את צמתי הווקטורים מהמסד.
5. **`StopEvent`** ➡️ מסנתז את התשובה הסופית תוך שימוש בחוקים קפדניים ומחזיר אותה לממשק.

*(תרשים ויזואלי של זרימת המידע מיוצר באופן אוטומטי כקובץ `workflow_design.html` בכל פעם שהמערכת רצה).*

## 🚀 התקנה והפעלה

### 1. שכפול הפרויקט (Clone)

```bash
git clone https://github.com/yk6749841-ai/AI-Rag-Project.git
cd ai_rag_project
 ```

### 2. התקנת תלויות (Dependencies)

```bash
pip install -r requirements.txt
```

### 3. הגדרת משתני סביבה
יש ליצור קובץ .env בתיקייה הראשית של הפרויקט ולהזין את מפתחות ה-API:

```text
COHERE_API_KEY=your_cohere_key
PINECONE_API_KEY=your_pinecone_key
```

### 4. הכנת הנתונים (הרצה חד-פעמית)
כדי לחלץ את הנתונים המובנים ממסמכי התיעוד (תיקיית /docs), יש להריץ:

```bash
python extract_data.py
```
פעולה זו תייצר את קובץ ה-extracted_data.json.

### 54. הפעלת האפליקציה

```bash
python app.py
```
ממשק המשתמש ייפתח בדפדפן בכתובת המקומית (לרוב: http://127.0.0.1:7860).
